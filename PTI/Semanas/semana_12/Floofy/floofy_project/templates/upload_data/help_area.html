O sistema de upload de dados via ficheiros aceita ficheiros .TXT e .CSV.

Por cada linha do ficheiro, os parâmetros têm que estar separados por ';' e sem espaços em branco.

Na área de upload de cada tipo de dados, aparece a lista de parâmetros com esta estrutura:

Exemplo: Upload de anos letivos

param1: nome

param2: data de início

param3: data de fim

Assim, uma linha aceitável seria:

"2021-2022;2021-09-15;2022-07-15"

No caso de o parâmetro ser um nome, basta indicar o nome, normalmente, em texto.

Se o parâmetro é uma data, tem que ter exatamente este formato: ano-mês-dia

Se o parâmetro é uma lista de algo, por exemplo uma lista de ID's, tem que ter

este formato: id1-id2-id3, cada elemento separado por "-".


Erros

No caso de haver um erro, é possível que a estrutura dos ficheiros esteja correta,

mas que haja algo que não corresponde às regras e à lógica da base de dados.

Exemplo1: tentar inserir um aluno numa disciplina que não existe.

Exemplo2: tentar inserir uma cadeira cujo código identificador já corresponde a outra.

É função do administrador conhecer a lógica da base de dados para evitar estes tipos de erros
